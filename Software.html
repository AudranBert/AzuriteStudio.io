Sport's pool optimizer

Java, JavaFX, Cplex, ​Object-oriented programming​, Team work
Download

This project lasted 1 year, during the first semester, we ​wrote the specifications according to the client's needs, we also chose how we will adress some difficulties etc. The second semester was the actual development. We needed to create an application in Java to organize pools for sports competitions. So it is divided into 3 big parts. First, you can add, change or remove teams. Each team is define by a sport, a city, a name and the category in that sport. They are stored in a database. Secondly, you can select which teams for which you want to make pools. Then, it uses an API to get the distances between the selected cities. And so, finally, it creates pools with the selected teams using the distances. The user can choose between 2 methods for creating them, a ​Greedy algorithm that will quickly find a solution, not the best one but a good one. The second method is a linear model, it finds the best solution but it takes way more time. Once, the user has the pools, he can show them in a graph like that, or export it to a CSV.
It was a very cool project because it had multiple very different features.

WebCrawler

Java, JavaFX, Multithreading​, Object-oriented programming​, Team work, Graphs, Progress reports

Download part1 Download part2

This project lasted 1 year, in the first semester we made the Web Crawler and in the second semester, we made tools to visualize and analyze it. The web crawler takes URLs as inputs and will start the crawling from them. You can select how far you want to go and how many threads it will use. Once finished, it generates a .graphml file which stores the result of the crawling. So in the second semester, we made an app that can load graphML and that can show the graph. It uses the Fruchterman-Reingold algorithm for placing the nodes. It uses the PageRank algorithm to decide the size of the nodes and it computes a bunch of other metrics like average distance, degrees...
Overall it is was a very interesting project and we sucessfully created the app.

VITKLIA

Python, Explicability, Clustering
See on GitHub

This is tool made during my bachelor internship, It helps visualizing projection of embeddings from speaker identification. The goal was to help explaining the predictions of a speaker identification model trained on the VoxCeleb dataset. These embeddings have often a lot of dimensions and so it is very difficult to visualize them. This toolkit can use LDA, PCA or UMAP to reduce in 2 or 3 dimensions. Then, you can plot the result. If you click on a point, it will play the sound associate to the embedding. Moreover, it computes prototypes: the utterance that is very well represented in the data (here the diamonds), and criticisms: the utterance that not well represented in the data (here the triangles). Each color on the plot is one speaker, if you click on a utterance it can also open a new plot with only that speaker.
This internship was my first experience with research and artificial intelligence and it confirmed in my choice of going into that field.

PDF Parser

Python, Scrum, Team work
See on GitHub

This project had a particular emphasis on the work organization. We made sure to apply Scrum the best we can. This tool allows to parse a PDF and transforms it to a XML file for example. Since PDFs have not real structure, sometimes, it will not find some parts of the article. The goal was to parse scientific articles automatically the best we could by searching for patterns in the files and using them to parse the files. Our parsers had good results and extracted titles, authors, introdution, etc, pretty well.
This was a good experience and a pretty intense project since we didn't had a lot of time to do it.
