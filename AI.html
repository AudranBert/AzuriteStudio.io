Physiav

Python, Pytorch, Speechbrain, Pandas, Numpy, Scikit-learn, Wav2Vec

During my apprenticeship at the LIA (Laboratoire Informatique d'Avignon), I worked on predicting emonotional and physiological state through the voice. But, there isn't a lot of data for that, so we used a pre-trained wav2vec called "LeBenchmark" (french finetuned wav2vec). We then finetuned again this model on our data. Physiav is a toolkit based on SpeechBrain (an AI toolkit for voice oriented tasks) that allows to finetune a wav2vec model, extract embeddings, transform data, train an embedding classifier/comparator, extracts features with Disvoice, reduce and plot embeddings. All of this on multiples datasets. There are recipes to do all of this, and a lot of classes and functions to be able to create our own recipes. The toolkit is accessible ony for the memebers of the LIA.
It was a very interesting work and I learnt a lot of things during it. 

Prediction of film review

Python, SVM, CamemBERT, data processing
See on github

This project's goal was to predict the rating associated to a review of a film. The data came from Allociné and there were over over 750k reviews in total on more than 3k films. We started by analyzing the datas to see any bias, or anything that could help in our task. Then, we made a basic SVM that we used for improving our data processing. At best with the SVM we had 67% of accuracy on 3 classes. We decided to take it further by using a language model. Since reviews were in french, we used CamemBERT. We used a pre-trained version that we finetuned on our data. The finetuning took a long time but at the end we had a huge improvement in 10 classes accuracy (from around 20% to 46%) and and almost 20% better on 3 classes. We also tried to add emojis to the prediction because we saw that some emojis were heavily correlated with the rating, but, since there are not used often, it didn't go far.
For me, it was the first time I used as many data and also my first time working on text so it was a very ​rewarding project.

BI on ​Insurance company

Python, Big data, data processing, kmeans...
Download report

We were a group of 3 for this project and the goal was to optimize the rendez-vous of a insurance company. Because it costs a lot of money to contact companies, therefore, getting a reject is wasting money. So, we must find informations that will increase the chance that the company will accept it. First, we analyzed the data to find correlation and know a bit more about the data. We also searched for any outliers or errors to decide what we should do with them. Then, we used unsupervised learning to try to find any clusters with anormal proportion of rendez-vous. We tested by looking at regions, department... Finally, we tried supervised learning technics like decision tree to see if again we can find any interesting conditions that has an anormal proportion of rendez-vous.

Explicability on speaker authentification

Python, Explainability, Interpretability, Speaker authentification
Download report

The goal of this project was to analyze the behavoir of a speaker authentification model. We needed to find the biais, the strenghts, the weaknesses of the model. One way to do that is to analyze some elements that were correctly classified but most importanly the elements that were not correctly classified. On the picture, I took all false positive and I removed parts of the audios until the model prediction changed. I divided audios into 50ms parts, and each time I removed the part that was the most influential on the decision of the model. By doing so, we can discover the biais. For example, I discovered that for one audio, it removed all the parts where the person was ​hesitant. The model was using hesitations to decide if it was the same person in both audios or not.
